📖 How to set up the Gmail label (step-by-step)

Rejection subjects are misleading. Most say “Thank you for applying…” in the subject — the actual rejection (“unfortunately…”, “after reviewing we've decided…”) is only inside the email body. Subject-based filters miss most of them, so the tracker reads the full email content to tell a real rejection from a mere acknowledgment.

Recommended — label you already use

You created job-rejection — perfect. Set that name in the field above (default). Any email under that label is imported; the tracker parses company + title from the body.

  1. When a rejection arrives, Gmail → select the message → click the Label icon → tick job-rejection.
  2. Or drag the email to the label in the left sidebar.
  3. In Job Tracker: connect rejections Gmail → Rej. → Fetch from Gmail.

Optional — auto-label new mail (broad filter)

This catches all application replies (including “we received your application”). The tracker still reads the body and only surfaces real rejections when not using the label — but with the label, everything tagged is shown for you to confirm.

  1. Gmail → Settings → Filters → Create a new filter.
  2. Has the words: 
    "thank you for applying" OR "update on your application" OR "your application" OR "interest in joining"
  3. Apply label job-rejection → tick Also apply filter to matching conversations.

No label yet? Fetch still works — the tracker searches your inbox and analyses each email's body for decision language (“unfortunately”, “after reviewing we've decided”, etc.).

📖 Setup guide

Step-by-step runbooks for every key this app can use. All of the recommended options (Gemini, Groq, JSearch, Jooble, Google Cloud) are free.

🤖 AI providers

Google Gemini free · recommended
  1. Open aistudio.google.com/app/apikey and sign in with any Google account.
  2. Click Create API key (pick “create in new project” if asked).
  3. Copy the key — it starts with AIza…
  4. Paste it in the AI provider tab → Gemini API key, set Provider to Google Gemini and save.

Why Gemini first: it's the only provider that supports web-grounded company research and voice transcription here, and the free tier covers normal daily use.

Groq free · fast
  1. Open console.groq.com/keys and sign up (Google or GitHub login works).
  2. Click Create API Key, give it any name.
  3. Copy the key — it starts with gsk_… (shown only once).
  4. Paste it in the AI provider tab → Groq API key. Great as a fallback provider even if Gemini stays primary.
OpenAI (GPT) paid
  1. Sign up at platform.openai.com (this is the API platform — separate from ChatGPT Plus).
  2. Add billing: Settings → Billing → Add payment details and load a small credit (e.g. $5) — new accounts have no free API credit.
  3. Go to API keys → Create new secret key.
  4. Copy the key (sk-…, shown only once) into the AI provider tab → OpenAI API key.
Anthropic (Claude) paid
  1. Sign up at console.anthropic.com.
  2. Buy prepaid credits under Billing (minimum ~$5).
  3. Go to API keys → Create key.
  4. Copy the key (sk-ant-…) into the AI provider tab → Anthropic API key.
Cursor needs a local proxy
  1. Get a key at cursor.com/dashboard → Integrations (crsr_…).
  2. Cursor has no native chat API, so a local OpenAI-compatible proxy must wrap the Cursor Agent CLI (e.g. cursor-agent-api-proxy).
  3. On this machine, start.command auto-starts the proxy on port 8080 whenever the provider is set to Cursor — usually nothing to do.
  4. Paste the key in the AI provider tab → Cursor API key and keep the base URL http://localhost:8080/v1.

🔎 Job search sources

JSearch / RapidAPI — LinkedIn listings free tier
  1. Sign up at rapidapi.com.
  2. Open the JSearch API page → Pricing → subscribe to the free Basic plan.
  3. Back on the API's Endpoints tab, copy the value of X-RapidAPI-Key from the code snippet on the right.
  4. Paste it in the Job sources tab → RapidAPI key.
Jooble free
  1. Open jooble.org/api/about and fill in the short request form.
  2. The API key arrives by email (usually within a day).
  3. Paste it in the Job sources tab → Jooble API key. The free quota is limited — the app tracks your remaining calls next to the field.
Web search over ATS job pages (Comeet, Greenhouse, Lever…) free · no key · works out of the box
  1. Nothing to set up. This source runs site: web searches (e.g. site:comeet.com/jobs automation Israel) through DuckDuckGo — no API key, no signup, no quota.
  2. The default sites list covers the ATS platforms most Israeli companies host jobs on: Comeet, Greenhouse, Lever, SmartRecruiters. Add or remove sites freely (comma-separated); clear the field to disable the source.
  3. Run a search from the Search page — each result is labeled with the board it came from: web:comeet, web:greenhouse, web:lever, web:smartrecruiters. Every link is opened before it is shown: closed postings and jobs outside your requested location are dropped automatically (takes a few extra seconds).

ℹ️ Why not Google? Google's Custom Search JSON API is closed to new customers (new projects get a permanent “project does not have access” 403, and the API retires in Jan 2027), so a Google Cloud key can't be used for this anymore. DuckDuckGo's results for site: queries are close to Google's for these job pages, with zero setup.

Adzuna optional · no Israel coverage
  1. Register at developer.adzuna.com and create an application.
  2. Copy the Application ID and Application Key shown on your dashboard.
  3. Paste both in the Job sources tab → Adzuna fields.

☁️ Google Cloud — one-time setup (Sheets sync + Gmail alerts)

Create the OAuth client JSON free · ~5 minutes · done once
  1. Create a project: open console.cloud.google.com, click the project picker (top bar) → New project → name it e.g. job-tracker → Create.
  2. Enable the APIs: go to APIs & Services → Library and enable, one by one: Google Drive API, Google Sheets API (for the sheet sync) and Gmail API (for job alerts).
  3. Consent screen: APIs & Services → OAuth consent screen → user type External → fill only app name + your email → save. The app can stay in Testing mode forever.
  4. Test users: on the consent screen page open Audience → Test users → + Add users and add every Google account you'll sign in with — e.g. your Sheets account, the alerts mailbox (ramizil1@gmail.com) and the rejections mailbox (zilber.rami@gmail.com). Without this, sign-in fails with “Access blocked” / שגיאה 403: access_denied.
  5. Create the client: APIs & Services → Credentials → + Create credentials → OAuth client ID → application type Desktop app → Create → Download JSON.
  6. Install it: save the downloaded file at the path shown in the Google tab → OAuth client JSON (default data/google_client_secret.json) — the status there should flip to “file found”.
  7. Connect: in the Google tab press Connect Google (Sheets) and/or Connect Gmail. A browser sign-in window opens; on the “Google hasn't verified this app” warning click Continue — it's your own app. Sheets and Gmail can use different accounts; each connection stores its own token.

Costs nothing at this usage level. Gmail access is requested read-only; the app can never send or delete mail. To revoke access later, use the Disconnect buttons or myaccount.google.com/permissions.
