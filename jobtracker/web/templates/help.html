🚀 Step-by-step: from zero to tailored CV

  1. Configure your keys (one time). Open ⚙ Settings and paste your API keys. Values save to your local .env and apply immediately.
  2. Add LinkedIn / job search. LinkedIn has no public jobs API, so jobs (including LinkedIn postings) come via JSearch. In Settings, add a RAPIDAPI_KEY from rapidapi.com → JSearch (free Basic plan → copy the X-RapidAPI-Key). Optionally add a free JOOBLE_API_KEY.
  3. Turn on AI. In Settings add a GEMINI_API_KEY from Google AI Studio to unlock fit analysis & resume tailoring.
  4. Point to your resume. Set RESUME_PATH in Settings to your CV (HTML), then click Rebuild match profile.
  5. Find jobs. Go to Search, enter a title (or leave blank to use your target titles) and location, and Save the ones you like. They land in your tracker as saved.
  6. …or add one manually. On Applications click + Add application, paste the LinkedIn URL and the job description (the description powers AI & match scoring).
  7. Track progress. Move applications through the pipeline (saved → applied → screening → interview → offer). Open an application to change its status, add notes, or mark it rejected (with stage + reason).
  8. Analyze fit. Open an application → Analyze fit with Gemini. You get a YES/MAYBE/NO verdict, a requirement-by-requirement match, risks, and concrete resume fixes.
  9. Tailor your CV. Tick the suggestions you want → Generate tailored resume. Review/edit it, then Print / Save as PDF (pixel-perfect) or Download PDF.
  10. Review & export. The Dashboard shows your funnel, response/interview rates and rejection reasons. Export everything via CSV / Excel in the top bar.

Starting & stopping the server

▶ Start the server

Easiest: double-click start.bat in the project folder (or JobTracker.vbs for no console window). A standalone app window opens automatically.

Or from PowerShell:

cd C:\GIT\RAMI\job-tracker
.\.venv\Scripts\Activate.ps1
python -m jobtracker web

Then open http://{{ host }} in your browser.

Options

# custom port
python -m jobtracker web --port 8080

# allow other devices on your network
python -m jobtracker web --host 0.0.0.0 --port 5001

■ Stop the server

Easiest: click ⏻ Quit in the top bar — or press Ctrl + C in the server console window.

If it is running in the background (or you closed the window), stop it by its port:

# find & stop whatever is listening on port {{ port }}
$p = (Get-NetTCPConnection -LocalPort {{ port }} -State Listen).OwningProcess
Stop-Process -Id $p -Force

Closing the terminal that launched it also stops it.

Restart

Stop it (above), then run the start command again.
